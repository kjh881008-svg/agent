### 🐍 2. 가상 환경 구축 및 실행 스크립트 (가이드)

가상 환경 구축은 OS별로 활성화(Activation) 방법이 완전히 다릅니다. 이 부분을 가장 헷갈려 하니, OS별로 명확하게 구분하여 가이드합니다.

---

#### 💡 [핵심 원칙] 왜 가상 환경을 써야 하나요?

시스템에 전역으로 패키지를 설치하면, 프로젝트 A가 요구하는 라이브러리 버전과 프로젝트 B가 요구하는 라이브러리 버전이 충돌할 수 있습니다. 가상 환경(`.venv`)은 프로젝트별로 격리된 독립적인 공간을 만들어, 패키지 버전 충돌(Dependency Hell)을 원천적으로 방지합니다. **이건 절대 건너뛰면 안 되는 단계입니다.**

#### 🚀 [단계별 가이드]

**1단계: 가상 환경 생성**

터미널에서 프로젝트 루트 디렉토리로 이동 후, 다음 명령어를 실행합니다.

```bash
# Python 3가 시스템에 설치되어 있다고 가정합니다.
python3 -m venv .venv
```

**2단계: 가상 환경 활성화 (가장 중요!)**

운영체제에 따라 실행 명령이 다릅니다.

| OS | 터미널 종류 | 명령어 |
| :--- | :--- | :--- |
| **macOS / Linux** | Bash / Zsh | `source .venv/bin/activate` |
| **Windows** | Command Prompt (cmd) | `.venv\Scripts\activate.bat` |
| **Windows** | PowerShell | `.venv\Scripts\Activate.ps1` |

> ⚠️ **확인 포인트:** 성공적으로 활성화되면, 터미널 프롬프트 앞에 `(.venv)`와 같은 가상 환경 이름이 붙어야 합니다.

**3단계: 필수 패키지 설치**

활성화된 가상 환경에서 다음 명령어를 실행하여 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

---

### 🐛 3. 발생 가능한 주요 오류 및 해결 가이드 (Troubleshooting)

이 부분이 실무에서 가장 중요합니다. 에러가 났을 때 "설치 안 돼요"가 아니라, "어떤 맥락에서 안 되는지"를 알아야 합니다.

| 오류 메시지 | 원인 분석 | 해결 방법 |
| :--- | :--- | :--- |
| `python: command not found` | 시스템에 Python 3가 설치되어 있지 않거나, PATH 설정이 안 됨. | OS별 설치 가이드를 따르거나, `pyenv` 같은 버전 관리 툴을 사용하세요. |
| `Scripts/activate: No such file or directory` | 2단계의 OS별 활성화 스크립트 경로를 잘못 지정함. | **반드시** 현재 디렉토리에 `.venv` 폴더가 존재하는지 확인하고, OS에 맞는 정확한 명령어를 사용하세요. |
| `Permission denied` | 가상 환경 생성/활성화 시 권한 문제가 발생함. | `sudo` 사용은 최후의 수단으로 남겨두고, 사용자 계정의 권한을 확인하거나 프로젝트 폴더의 권한을 재설정하세요. |
| `Requirement already satisfied` | 이미 설치된 경우입니다. (에러 아님) | 정상입니다. 패키지 목록을 확인하는 것으로 충분합니다. |

### ✅ 4. 실행 검증 (Verification)

```bash
pip freeze

이 명령어를 실행했을 때, `fastapi`, `uvicorn`, `sqlalchemy` 등이 목록에 포함되어야 합니다.

from sqlalchemy import Column, Integer, String, DateTime, func
from sqlalchemy.ext.declarative import declarative_base

# SQLAlchemy의 기본 베이스 클래스 정의
Base = declarative_base()

class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True, index=True)
    username = Column(String, unique=True, index=True, nullable=False)
    email = Column(String, unique=True, index=True, nullable=False)
    hashed_password = Column(String, nullable=False)
    created_at = Column(DateTime, default=func.now())

class Content(Base):
    __tablename__ = "contents"
    id = Column(Integer, primary_key=True, index=True)
    title = Column(String, index=True, nullable=False)
    author_id = Column(Integer, index=True) # Foreign Key: User.id
    body = Column(String, nullable=False)
    is_published = Column(Integer, default=0) # 0: 비공개, 1: 공개
    created_at = Column(DateTime, default=func.now())

    # TODO: 실제 비밀번호 해싱 로직 (passlib 등) 추가 필요

### 🛠️ 1순위 작업 착수: GitHub Actions CI/CD 워크플로우 설정

CI/CD를 구현하려면, 프로젝트 루트에 `.github/workflows/` 디렉토리를 만들고 YAML 파일을 작성해야 합니다. 이 워크플로우는 `main` 브랜치에 푸시되거나 PR이 열릴 때마다 자동으로 실행되어, 개발자가 실수로 불안정한 코드를 메인 브랜치에 올리는 것을 막아주는 방어막 역할을 합니다.

**⚠️ 코다리 검증 포인트:**
1.  Python 환경 설정 및 가상 환경 구축 과정을 명시했습니다.
2.  `requirements.txt`를 통해 필요한 모든 의존성을 한 번에 설치하도록 했습니다.
3.  `pytest`를 통해 테스트 실행을 강제합니다.
4.  Linting 단계도 포함하여 코드 스타일까지 검증합니다.

먼저 파일 구조를 잡겠습니다.

<create_file path=".github/workflows/ci_cd.yml">
name: CI/CD Pipeline

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build_and_test:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout repository
      uses: actions/checkout@v3

    - name: Set up Python
      uses: actions/setup-python@v3
      with:
        python-version: '3.10' # Python 3.10 안정화 버전 사용

    - name: Install dependencies
      run: |
        python -m venv venv
        source venv/bin/activate
        pip install -r requirements.txt
        # FastAPI와 SQLAlchemy 등 추가 라이브러리 설치가 필요할 수 있습니다.
        # pip install "fastapi[all]" python-jose[cryptography] passlib[bcrypt]

    - name: Run Unit Tests
      run: |
        source venv/bin/activate
        # pytest를 사용하여 테스트를 실행합니다.
        # 테스트 환경 변수 설정이 필요할 수 있습니다.
        pytest

    - name: Linting Check (Optional)
      run: |
        source venv/bin/activate
        # flake8 또는 black 등을 사용하여 코드 스타일을 검사합니다.
        flake8 src/
</files>
