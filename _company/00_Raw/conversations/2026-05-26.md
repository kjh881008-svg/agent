```yaml
name: Python CI/CD Pipeline

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build_and_test:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout code

    - name: Set up Python

    - name: Install dependencies

        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run Linters and Format Checks (Optional)
      run: |
        # flake8 또는 black 같은 린터를 사용합니다.
        echo "Running code quality checks..."
        # flake8 .

    - name: Run Unit Tests
      # 프로젝트 루트 디렉토리에서 pytest를 실행합니다.

        pytest
```

```markdown
## CI/CD 파이프라인

    1. 코드를 체크아웃합니다.

이 파이프라인이 성공적으로 완료되어야만 `main` 브랜치에 병합(Merge)될 수 있습니다.
```

### 🛠️ 1단계: 문제 원인 분석 (가설 설정)

`auto_planner.py`가 FastAPI 서버와 호환성 문제를 겪는 가장 흔한 원인은 다음과 같습니다.

1.  **Context Leakage**: 스크립트가 FastAPI의 Request/Response 객체나 DB Session 같은 서버 컨텍스트에 의존하는데, 이 컨텍스트가 실행 환경(예: CI/CD 스크립트)에서 주입되지 않은 경우.
2.  **환경 변수 누락**: API 키나 다른 설정값이 서버 레벨에서 로드되어야 하는데, 스크립트 레벨에서 로드되지 않은 경우.
3.  **비동기 처리 문제**: FastAPI가 `asyncio`를 기반으로 동작하는데, 스크립트 내의 일부 함수가 비동기적으로 처리되지 않은 경우.

일단, 코드를 직접 볼 수 없으므로, 가장 일반적인 **Context 의존성 문제**를 해결하는 방향으로 접근하겠습니다. 만약 `auto_plans` 같은 함수가 요청 객체(`request`)에 의존한다면, 이 함수를 독립적으로 만들거나, 필요한 의존성(Dependencies)을 함수 인자로 받도록 수정해야 합니다.

#### 2. `src/logic/user_logic.py` 신규 생성 (핵심 분리)

순수한 계산 로직을 담는 새로운 모듈을 생성합니다. 이 모듈은 어떤 DB나 HTTP 클라이언트에도 의존하지 않습니다.

```python
# src/logic/user_logic.py (신규 파일)

def calculate_user_score(points: int, activity_days: int) -> int:
    """
    사용자의 포인트와 활동 일수를 기반으로 사용자 점수를 계산합니다.
    이 함수는 순수한 수학적 로직만 포함합니다.
    """
    if activity_days < 7:
        return points * 1.1  # 활동량이 적으면 가산점 부여
    else:
        return points + (activity_days * 5)
```

#### 3. 데이터베이스 클라이언트 인터페이스 정의 (선택적 개선)

강한 결합을 피하기 위해, `DBClient`에 대한 추상 클래스(ABC)를 정의하는 것이 가장 이상적입니다.

```python
# src/database/db_client.py (개선 방향)
from abc import ABC, abstractmethod

class AbstractDBClient(ABC):
    @abstractmethod
    def fetch_user_data(self, user_id: int) -> dict | None:
```
