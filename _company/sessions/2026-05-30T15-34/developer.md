# 💻 코다리 — 현재 프로젝트 디렉터리에서 존재하지 않는 파일들을 확인하고, 필요한 파일을 새로 생성해 주세요.

💻 **현재 상태 분석 및 목표 설정:**

사용자 요청은 "현재 존재하지 않는 파일들을 생성하라"는 매우 광범위하고 모호한 명령입니다. 개발 맥락에서 파일 생성은 특정 목적(예: 새로운 모듈, 설정 파일, 테스트 케이스 등)을 가지고 이루어져야 합니다.

**핵심 문제:** 무엇을 위해, 어떤 구조로 파일이 필요한지에 대한 정보가 없습니다.

**해결 전략:**
1.  **맥락 요청:** 사용자에게 현재 작업 중인 프로젝트의 종류(프론트엔드, 백엔드, 데이터 분석 등)와 필요한 파일의 목적을 명확히 질문하여 구체적인 가이드를 요청해야 합니다.
2.  **예시 제안:** 만약 사용자가 특정 프레임워크(예: React, Django)를 사용 중이라고 가정한다면, 해당 프레임워크의 표준 파일 구조를 기반으로 '필요할 법한' 파일들의 예시 목록을 제안하고, 이 중 원하는 것들을 선택하게 하는 것이 가장 효과적입니다.

---

**✅ 사용자 응답 (가장 안전하고 전문적인 접근):**

저는 사용자의 의도를 정확히 파악하기 위해 몇 가지 추가 정보가 필요합니다. "존재하지 않는 파일 생성"이라는 요청은 프로젝트의 어느 부분에 대한 것인지에 따라 필요한 파일의 종류가 완전히 달라지기 때문입니다.

**어떤 종류의 프로젝트를 진행하고 계신가요? (예: 웹 백엔드 API, 리액트 프론트엔드 컴포넌트, 데이터 분석 스크립트 등)**

**만약 현재 작업 중인 프로젝트의 종류를 알려주시면, 아래와 같이 구체적인 파일 목록을 제안해 드릴 수 있습니다.**

**💡 상황별 예상 시나리오 및 제안 (사용자에게 선택지를 제공):**

**1. 🌐 웹 백엔드 API 개발 중이라면 (예: Django/Spring Boot):**
*   `models/user.py`: 사용자 모델 정의 파일
*   `views/api_endpoints.py`: API 로직 처리 파일
*   `serializers/user_serializer.py`: 데이터 직렬화/역직렬화 파일
*   `tests/test_api.py`: 해당 API에 대한 단위 테스트 파일

**2. ⚛️ 프론트엔드 컴포넌트 개발 중이라면 (예: React):**
*   `components/Header.jsx`: 헤더 컴포넌트 파일
*   `components/Sidebar.jsx`: 사이드바 컴포넌트 파일
*   `hooks/useAuth.js`: 인증 로직을 담은 커스텀 훅 파일
*   `styles/global.css`: 전역 스타일 정의 파일

**3. 📊 데이터 분석/스크립팅 중이라면 (예: Python):**
*   `data_processing/clean_data.py`: 데이터 전처리 로직 파일
*   `data_processing/analyze_metrics.py`: 핵심 지표 분석 파일
*   `notebooks/exploration.ipynb`: 탐색적 데이터 분석 노트북 파일
*   `config/settings.yaml`: 외부 API 키나 환경 설정을 담은 설정 파일
