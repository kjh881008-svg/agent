## 🛠️ 프로젝트 환경 구축 및 실행 가이드

### 🟢 Step 0: 개발 환경 격리 (Virtual Environment 구축)

**[원칙]** 프로젝트마다 필요한 라이브러리 버전이 다를 수 있습니다. 전역(Global) 환경에 직접 패키지를 설치하면 다른 프로젝트와 충돌할 위험이 매우 높습니다. 따라서, **가상 환경(Virtual Environment)**을 만들어 프로젝트에 필요한 라이브러리만 격리하는 것이 필수입니다.

**💻 실행 명령어:**

```bash
# 1. 가상 환경 생성 (프로젝트 폴더 내에 'venv'라는 폴더 생성)
python3 -m venv venv

# 2. 가상 환경 활성화 (macOS/Linux 기준)
source venv/bin/activate

# 2. 가상 환경 활성화 (Windows 기준)
# .\venv\Scripts\activate
```

> 💡 **확인:** 명령 프롬프트 앞에 `(venv)`와 같이 가상 환경 이름이 보이면 성공적으로 활성화된 것입니다.

### 🟡 Step 1: 의존성 설치 (Dependencies Installation)

**[원칙]** 프로젝트가 필요로 하는 모든 외부 라이브러리(패키지)를 한 번에 설치합니다. 보통 이 목록은 `requirements.txt` 파일에 작성되어 있습니다.

**💻 실행 명령어:**

```bash
# requirements.txt 파일에 명시된 모든 라이브러리를 설치합니다.
pip install -r requirements.txt
```

### 🟡 Step 2: 환경 변수 설정 (Configuration: API Keys 관리)

**[원칙]** API 키, 데이터베이스 비밀번호 등 민감한 정보(Secret Key)는 코드(`*.py`) 안에 직접 하드코딩해서는 절대 안 됩니다. 보안을 위해 **환경 변수(Environment Variables)**를 사용하며, 이를 관리하는 표준 파일이 `.env` 파일입니다.

**💻 실행 명령어 (파일 생성 및 내용 작성):**

1.  프로젝트 루트 디렉토리에 `.env` 파일을 생성합니다.
2.  파일 내부에 다음과 같은 형식으로 필요한 키를 입력합니다.

    ```dotenv
    # .env 파일 내용 예시
    API_KEY="여기에실제_API_키를_입력하세요"
    DATABASE_URL="postgresql://user:password@host:port/dbname"
    SECRET_KEY="매우_긴_랜덤_문자열"
    ```

> ⚠️ **주의:** `.env` 파일은 절대 Git 저장소에 커밋하면 안 됩니다. 반드시 `.gitignore` 파일에 추가하여 업로드되지 않도록 처리해야 합니다.

### 🟢 Step 3: 스크립트 실행 (Execution)

**[원칙]** 모든 준비가 끝났으므로, 메인 스크립트를 실행합니다. 만약 스크립트가 여러 단계를 거쳐야 한다면, 실행 스크립트(예: `run.py`)를 통해 호출하는 것이 좋습니다.

**💻 실행 명령어:**

```bash
# 메인 애플리케이션 스크립트를 실행합니다.
python main.py
```

## 🔍 문제 해결 및 추가 팁 (Troubleshooting & Tips)

| 문제 상황 | 원인 분석 | 해결 방안 |
| :--- | :--- | :--- |
| `ModuleNotFoundError` 발생 | 필요한 라이브러리가 설치되지 않았거나, 가상 환경이 활성화되지 않았을 가능성이 높습니다. | **Step 0**부터 다시 실행하고, `pip install -r requirements.txt`를 다시 실행하세요. |
| 키나 비밀번호가 인식되지 않음 | `.env` 파일이 존재하지 않거나, 코드가 환경 변수를 로드하는 라이브러리(예: `python-dotenv`)를 사용하지 않았을 수 있습니다. | **Step 2**를 다시 확인하고, 코드가 `.env` 파일을 읽도록 로직을 추가해야 합니다. |
| `pip` 명령어 오류 | 시스템에 Python 버전이 여러 개 설치되어 충돌이 발생할 수 있습니다. | **`python3`** 명령어를 명시적으로 사용하거나, 가상 환경을 사용하는 것이 가장 안전합니다. |

이 가이드대로 진행하시면 안정적인 개발 환경이 구축될 것입니다. 추가적인 궁금증이나 특정 오류가 발생하면 언제든지 다시 질문해주세요!
